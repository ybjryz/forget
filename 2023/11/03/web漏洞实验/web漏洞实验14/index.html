0x00 漏洞搭建

进入根目录下

docker起环境

docker-compose up -d

docker ps 检查

1.突破功能限制漏洞

要求突破查询按钮disable限制，获取编号：110010的查询内容（弹框中的flag）

f12 删除disable属性

2.用户信息泄漏漏洞

通过回显信息，以暴力破解方式猜测系统中存在的两个用户名(提示：u 开头)，并暴力破解这两个用户对应的密码

burp抓包导入intruder

猜到一个为user 找了一本字典来跑 跑出密码为888888

另一个用户为user1

同样操作 跑出结果为123456

3. 越权漏洞

通过上一步破解的用户名密码登录，利用越权漏洞获取admin1的个人信息。

学习一下伪造的思路

分析一下抓到的包

cookie中的uid 经过url+base64解码是user

因此思路明了了 把user换成admin1 用相同的编码方式

得到admin1的信息

4. 遍历漏洞

已知当前系统中存在的一个学号为：20190504035XA01。利用遍历漏洞获取其他学号对应的成绩（不少于5条）

burp抓包后发送到intruder

采用集束炸弹攻击

集束炸弹攻击：理解为排列组合中的乘法 依次使用每一个payload

为了显示方便 使用sqlmap将结果显示出来

5.暴力破解攻击

获取系统中另一个六位数字编号对应的弹框flag内容

和1一样 把1的包内容加上payload

设置payload

和1一样 删除disable属性
